# 🌍 AirSense India — Air Quality Analysis & Green Solution System

A professional data science project analyzing air pollution across **26 Indian cities (2015–2020)**, with an interactive Streamlit dashboard, ML-based AQI prediction, and plant-based remediation recommendations.

---

## 📁 Project Structure

```
Air Pollution/
│
├── app.py                # Main Streamlit dashboard (4 pages)
├── train_model.py        # One-time script to train & save the ML model
├── model.pkl             # Pre-trained Random Forest model (generated by train_model.py)
├── cleaned_data.csv      # Cleaned dataset (24,850 records × 18 features)
├── city_day.csv          # Raw dataset (29,531 records × 16 features)
├── EDA_ML.ipynb          # Exploratory data analysis & ML notebook
└── requirements.txt      # Python dependencies
```

---

## 🚀 Quick Start

### 1. Clone / open the project folder
```bash
cd "Air Pollution"
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Train the model (run once)
```bash
python train_model.py
```
This generates `model.pkl` so the app loads instantly on every subsequent launch.

### 4. Launch the dashboard
```bash
streamlit run app.py
```

## 🤖 ML Model

- **Algorithm:** Random Forest Classifier (120 trees, class-weighted)
- **Features:** PM2.5, PM10, NO, NO2, NOx, NH3, CO, SO2, O3, Benzene, Toluene
- **Target:** AQI Bucket (Good / Satisfactory / Moderate / Poor / Very Poor / Severe)
- **Accuracy:** ~81% on held-out test set
- **Handles class imbalance** via `class_weight='balanced'`

## 🗃️ Dataset

**Source:** Central Pollution Control Board (CPCB), India  
**Coverage:** 26 cities · 2015–2020 · Daily readings  

| File | Rows | Columns | Notes |
|------|------|---------|-------|
| `city_day.csv` | 29,531 | 16 | Raw data with nulls |
| `cleaned_data.csv` | 24,850 | 18 | Cleaned + `Year`, `Month`, `Season` added |

**Cleaning steps applied in `EDA_ML.ipynb`:**
- Dropped `Xylene` column (>60% missing)
- City-wise median imputation for remaining nulls
- Dropped rows with missing AQI values
- Outlier capping at 1st–99th percentile
- Added `Year`, `Month`, `Season` from `Date` column

## 🌿 Plant Recommendation Logic

The app computes the **dominant pollutant** per city dynamically by finding which pollutant has the highest correlation with AQI in that city's data. It then maps that pollutant to scientifically-backed plant species known to absorb it.

| Pollutant | Example Plants |
|-----------|---------------|
| PM2.5 | Bamboo, Neem, Areca Palm |
| PM10 | Peepal, Ashoka, Holy Basil |
| CO | Snake Plant, Money Plant, Spider Plant |
| NO2 | Eucalyptus, Silver Oak, Gulmohar |
| SO2 | Cassia, Marigold, Bougainvillea |
| O3 | Banyan, Rain Tree, Rubber Plant |
| Benzene | Peace Lily, Gerbera Daisy, Chrysanthemum |

> **Note:** The AQI reduction simulation (3% per 10% green cover increase) is a conservative estimate based on urban greening literature and should be treated as indicative, not prescriptive.

## 📝 Notes

- The app uses `@st.cache_data` and `@st.cache_resource` so data and the model are loaded only once per session.
- If `model.pkl` is missing, the app automatically falls back to retraining on startup (takes ~30 seconds).
- All charts are built with Plotly for interactivity (hover, zoom, pan).
